## Работа с новыми файлами cats.txt и dogs.txt

### Предварительные условия
1. **Запустите ChromaDB** (если еще не запущена):
   ```bash
   docker run -d -p 8000:8000 --name chromadb chromadb/chroma
   ```

2. **Запустите Ollama** с нужными моделями:
   ```bash
   ollama pull phi3:mini
   ollama pull nomic-embed-text
   ollama serve
   ```

3. **Запустите MCP сервер McpRag**:
   ```bash
   cd McpRag
   dotnet run
   ```

### Шаг 1: Индексация документов
Сначала нужно проиндексировать папку с файлами, чтобы их содержимое попало в векторное хранилище:

**Через Cline:**
```
Вызови инструмент index_folder с folderPath "./test_docs"
```

**Или с указанием паттерна:**
```
Вызови инструмент index_folder с folderPath "./test_docs" и pattern "*.txt"
```

**Ожидаемый результат:**
```
Загружено 5 файлов, общий размер 150 символов
```

### Шаг 2: Проверка загруженных файлов
Можно проверить, что файлы загрузились:

**Через Cline:**
```
Вызови инструмент list_files
```

**Ожидаемый результат:**
```
cats.txt (0.1 KB, 2024-01-01), dogs.txt (0.1 KB, 2024-01-01), file1.txt (1.2 KB, 2024-01-01), file2.txt (0.9 KB, 2024-01-01), large.txt (2.1 KB, 2024-01-01)
```

### Шаг 3: Проверка статуса векторного хранилища
Убедитесь, что документы добавлены в ChromaDB:

**Через Cline:**
```
Вызови инструмент vector_store_status
```

**Ожидаемый результат:**
```
✅ Vector store status:
- Type: chromadb
- Server: http://localhost:8000
- Collection: documents
- Documents: 15
```

### Шаг 4: Задавайте вопросы по содержимому файлов

Теперь можно задавать вопросы по содержимому файлов cats.txt и dogs.txt:

#### Примеры вопросов:

1. **Про кошек:**
   ```
   Вызови инструмент search_docs с query "самая красивая кличка для кошки"
   ```

2. **Про Моню:**
   ```
   Вызови инструмент search_docs с query "Моня кошка"
   ```

3. **Про полосатых кошек:**
   ```
   Вызови инструмент search_docs с query "полосатые кошки умеют летать"
   ```

4. **Про собак:**
   ```
   Вызови инструмент search_docs с query "самая красивая кличка для собаки"
   ```

5. **Про Рекса:**
   ```
   Вызови инструмент search_docs с query "Рекс собака"
   ```

### Шаг 5: Использование RAG для ответов

Если вы хотите получить ответы на основе содержимого файлов (не просто поиск, а генерация ответов), нужно использовать инструмент `ask_question` (если он реализован в вашей версии):

```
Вызови инструмент ask_question с question "Какая самая красивая кличка для кошки?"
```

### Важные моменты

1. **Индексация выполняется один раз** - после индексации файлы остаются в векторном хранилище
2. **При изменении файлов** нужно перезапустить индексацию
3. **Для очистки хранилища** используйте: `Вызови инструмент clear_vector_store`
4. **Все запросы выполняются через Cline** - не нужно запускать команды в терминале
