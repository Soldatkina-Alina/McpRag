✅ Задача 11: Retry-цикл с переписыванием запроса
Что тестируем: При недостатке документов запрос переписывается и расширяется (макс 2 попытки)

Тест 1: Сложный вопрос (нужен retry)
Подготовь документы с синонимами:

bash
echo "Асинхронное программирование в C# использует async/await" > test_docs/async.txt
echo "Метод с async может содержать await для ожидания Task" > test_docs/async2.txt 
Запрос с плохой формулировкой:

text
"Спроси: как делать несколько дел одновременно не блокируя поток"
✅ Должен сработать retry:

Сначала RewriteQuery улучшает запрос

Если мало результатов → BroadenQuery расширяет

Путь должен быть: RewriteQuery → Search → GradeDocuments → (мало) → BroadenQuery → Search → GradeDocuments → Generate

Тест 2: Проверка лимита попыток
Запрос по теме, которой вообще нет в документах:

text
"Спроси: как приготовить борщ"
✅ Должен сделать 2 попытки (макс retry), потом MaxRetriesReached
В метаданных должно быть retry_count: 2

Тест 3: История запросов
После любого сложного вопроса в ответе должна быть секция:

text
📝 История запросов:
1. как делать несколько дел одновременно не блокируя поток
2. асинхронное программирование C# async await неблокирующее выполнение
3. многозадачность параллельное выполнение C#
Тест 4: Динамический порог
При каждой retry порог релевантности должен снижаться (например, 0.7 → 0.63 → 0.56)
В метаданных SearchNode видно threshold на каждом шаге
