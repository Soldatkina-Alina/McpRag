# Задача 11: Реализация Retry-цикла с перезаписью и расширением запроса

## Описание задачи
Добавить в граф состояний RAG retry-цикл с возможностью перезаписи и расширения поискового запроса.

## Выполненные шаги

1. **Обновление RAGConfig**
   - Добавлен класс `RetryConfig` с параметрами для управления retry-циклом
   - Добавлено свойство `Retry` в `RAGConfig`

2. **Расширение RagState**
   - Добавлены поля для хранения текущего запроса, истории запросов и динамического порога релевантности
   - Добавлено свойство для подсчета количества попыток поиска

3. **Реализация новых узлов графа**
   - `RewriteQueryNodeAsync`: перезаписывает запрос с помощью LLM для улучшения поиска
   - `BroadenQueryNodeAsync`: расширяет запрос с помощью LLM при недостатке релевантных документов
   - `HasEnoughRelevantDocuments`: проверяет, достаточно ли релевантных документов

4. **Обновление логики графа**
   - Добавлен retry-цикл с проверкой достаточности релевантных документов
   - Динамическое снижение порога релевантности при каждой попытке
   - Обработка лимита максимальных попыток

5. **Обновление SearchNodeAsync**
   - Использование текущего запроса (возможно, переписанного или расширенного) вместо исходного вопроса

6. **Обновление AskQuestionTool**
   - Добавление отображения истории запросов в результатах

7. **Обновление appsettings.json**
   - Добавление секции конфигурации для retry-цикла

## Результат

Внедрена функциональность для автоматического улучшения поискового запроса при недостатке релевантных документов. Система теперь:
- Переписывает запрос с помощью LLM перед первым поиском
- Расширяет запрос при необходимости с учетом истории
- Повторяет поиск до достижения нужного количества релевантных документов или максимального числа попыток
- Отображает историю запросов в результатах
