# Задача 9: Базовая структура графа состояний

## Описание
В этой задаче реализована базовая структура графа состояний для системы RAG (Retrieval-Augmented Generation). Граф состояний позволяет более гибко управлять процессом обработки запросов, разбив его на отдельные узлы с четкими ответственными функциями.

## Изменения в существующих файлах
- **McpRag/Tools/AskQuestionTool.cs**: Обновлен для использования сервиса графа состояний вместо прямого вызова методов.
- **McpRag/Program.cs**: Добавлена регистрация сервиса графа состояний в DI контейнере.

## Новые файлы
### McpRag/ExecutionStep.cs
Класс, представляющий шаг выполнения графа RAG. Содержит:
- `NodeName`: Имя узла графа
- `Timestamp`: Временная метка выполнения шага
- `Metadata`: Метаданные о выполнении шага (количество найденных чанков, релевантность и т.д.)

### McpRag/RagState.cs
Класс, представляющий состояние графа RAG. Содержит:
- `Question`: Вопрос пользователя
- `Documents`: Найденные документы
- `Answer`: Ответ на вопрос
- `ExecutionSteps`: Шаги выполнения графа
- `HasError`: Флаг наличия ошибки
- `ErrorMessage`: Сообщение об ошибке
- `HasRelevantDocuments`: Вспомогательное свойство для проверки наличия релевантных документов

### McpRag/IRagGraphService.cs
Интерфейс для сервиса графа RAG. Определяет метод:
- `ExecuteAsync`: Выполняет график RAG для получения ответа на вопрос

### McpRag/RagGraphService.cs
Реализация сервиса графа RAG. Содержит:
- `SearchNodeAsync`: Узел для поиска документов в векторном хранилище
- `GenerateNodeAsync`: Узел для генерации ответа с помощью LLM
- `ExecuteAsync`: Основной метод для выполнения графа

## Запуск и тестирование
1. Убедитесь, что Docker запущен и контейнер с ChromaDB активен
2. Убедитесь, что Ollama запущено и модели `phi3:mini` и `nomic-embed-text` установлены
3. Постройте проект с помощью команды: `cd McpRag; dotnet build`
4. Запустите сервер: `cd McpRag; dotnet run`
5. Подключитесь к серверу через MCP Inspector и протестируйте инструмент `ask_question`
