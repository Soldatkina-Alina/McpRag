# Задача 9: Базовая структура графа состояний для RAG

## Описание задачи
Задача 9 направлена на создание базовой структуры графа состояний для системы RAG (Retrieval-Augmented Generation), которая позволяет более гибко и управляемым образом обрабатывать вопросы пользователей. График состояний заменяет прямой подход на этапы обработки, делая систему более расширяемой и поддерживаемой.

## Выполненные шаги

### 1. Обновление структуры данных
- **DocumentChunk**: Добавлено поле `Score` для хранения релевантности фрагмента документа
- **VectorStoreServiceExtensions**: Изменено начальное значение Score с 0.8f на 0.0f для более точной проверки порога релевантности

### 2. Создание классов для графа состояний
- **ExecutionStep**: Класс для представления отдельного шага выполнения графа
- **RagState**: Класс для хранения состояния графа во время выполнения запроса
- **IRagGraphService**: Интерфейс для сервиса графа RAG
- **RagGraphService**: Реализация сервиса графа RAG с основными узлами:
  - SearchNodeAsync: Поиск релевантных документов в ChromaDB
  - GenerateNodeAsync: Генерация ответа с помощью LLM

### 3. Обновление инструментов
- **AskQuestionTool**: Переработан для использования сервиса графа RAG вместо прямого вызова методов
- **Program.cs**: Добавлена регистрация сервиса графа RAG в DI контейнере

### 4. Улучшение логики проверки релевантности
В `RagGraphService` улучшена проверка релевантности документов:
```csharp
if (!state.HasRelevantDocuments || state.Documents.Count == 0 || state.Documents.All(d => d.Score < _config.Value.MinRelevanceScore))
```
Это гарантирует, что запрос будет отклонен, если:
- Нет релевантных документов
- Нет документов в результате поиска
- Все документы имеют релевантность ниже порога

## Результат
Создна базовая структура графа состояний для RAG системы, которая:
1. Делит процесс обработки на отдельные узлы
2. Логирует каждый шаг выполнения
3. Улучшает проверку релевантности документов
4. Возвращает детальную информацию о результатах
5. Поддерживает конфигурацию параметров

Система готова для дальнейшего расширения с добавлением новых узлов графа (например, оценка документов, перезапросы, проверка галлюцинаций и т.д.)
