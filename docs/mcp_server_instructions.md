# Инструкция по работе с McpRag сервером

## Важные ограничения

**НИКОГДА не запускайте McpRag сервер вручную через `dotnet run`!**

### Проблема
Каждый раз, когда вы пытаетесь запустить McpRag сервер через команду `dotnet run`, это приводит к:
- Конфликту с уже работающим сервером, который управляется через Cline
- Блокировке файла McpRag.exe
- Ошибкам сборки и невозможности использовать сервер
- Поломке MCP сервера

### Правильный подход
1. **Не трогайте McpRag сервер** - он уже работает через Cline
2. **Все изменения в коде** должны применяться без перезапуска сервера
3. **Проверка изменений** должна осуществляться через существующее подключение к серверу через Cline
4. **Перезапуск сервера** (если действительно необходим) должен выполняться ТОЛЬКО через настройки Cline, а не вручную

### Как работать с сервером
- Вносите изменения в код файлов
- Не перезапускайте сервер вручную
- Используйте существующее подключение к McpRag через Cline для проверки изменений
- Если сервер действительно нужно перезапустить, делайте это через настройки Cline

### Команды, которые НЕЛЬЗЯ использовать
```bash
cd McpRag
dotnet run  # ❌ НЕЛЬЗЯ!
```

### Команды, которые можно использовать
```bash
# Только для чтения файлов и анализа кода
read_file McpRag/OllamaService.cs
search_files McpRag "*.cs" "IsModelAvailableAsync"
# и т.д.
```

**Соблюдение этих правил предотвратит поломку MCP сервера и обеспечит стабильную работу.**

## Правила для работы с MCP сервером при поиске ответов

### Основные команды MCP сервера и их назначение
- `index_folder` - индексирует содержимое папки для последующего поиска
- `search_docs` - поиск релевантных документов в векторном хранилище по текстовому запросу
- `list_files` - вывод списка загруженных файлов с метаданными
- `ask_llm` - прямой запрос к LLM (Large Language Model)
- `ask_question` - вопрос с поиском в векторном хранилище + генерация ответа через LLM
- `echo` - тестирование подключения к серверу
- `vector_store_status` - проверка статуса векторного хранилища
- `check_ollama` - проверка доступности Ollama и нужных моделей

### Правила для запросов с тегом [M]
1. Если запрос содержит тег [M], это значит, что нужно использовать инструмент MCP сервера
2. Необходимо найти подходящую функцию у MCP сервера и вызвать ее
3. Если MCP сервер не доступен или вернул ошибку, нужно написать об этом
4. Никогда не пытайтесь запустить MCP сервер самостоятельно
5. Всегда используйте существующее подключение к серверу через Cline

### Примеры использования
**Запрос:** [M] Какие кошки умеют летать?
**Действие:** Вызовите инструмент `ask_question` или `search_docs` с query "какие кошки умеют летать"

**Запрос:** [M] Список загруженных файлов
**Действие:** Вызовите инструмент `list_files`

**Запрос:** [M] Проверить доступность Ollama
**Действие:** Вызовите инструмент `check_ollama`

### Ошибка подключения к серверу
Если MCP сервер недоступен (например, ошибка "Request timed out"), просто сообщите об этом, не пытаясь запустить сервер самостоятельно.

**Соблюдение этих правил обеспечит эффективное и безопасное использование MCP сервера для получения ответов.**
