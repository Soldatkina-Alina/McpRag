# Инструкция по работе с McpRag сервером

## Важные ограничения

**НИКОГДА не запускайте McpRag сервер вручную через `dotnet run`!**

### Проблема
Каждый раз, когда вы пытаетесь запустить McpRag сервер через команду `dotnet run`, это приводит к:
- Конфликту с уже работающим сервером, который управляется через Cline
- Блокировке файла McpRag.exe
- Ошибкам сборки и невозможности использовать сервер
- Поломке MCP сервера

### Правильный подход
1. **Не трогайте McpRag сервер** - он уже работает через Cline
2. **Все изменения в коде** должны применяться без перезапуска сервера
3. **Проверка изменений** должна осуществляться через существующее подключение к серверу через Cline
4. **Перезапуск сервера** (если действительно необходим) должен выполняться ТОЛЬКО через настройки Cline, а не вручную

### Как работать с сервером
- Вносите изменения в код файлов
- Не перезапускайте сервер вручную
- Используйте существующее подключение к McpRag через Cline для проверки изменений
- Если сервер действительно нужно перезапустить, делайте это через настройки Cline

### Команды, которые НЕЛЬЗЯ использовать
```bash
cd McpRag
dotnet run  # ❌ НЕЛЬЗЯ!
```

### Команды, которые можно использовать
```bash
# Только для чтения файлов и анализа кода
read_file McpRag/OllamaService.cs
search_files McpRag "*.cs" "IsModelAvailableAsync"
# и т.д.
```

### Для перезапуска сервера
Только через настройки Cline:
1. Откройте настройки Cline
2. Найдите конфигурацию McpRag сервера
3. Перезапустите сервер через интерфейс Cline

**Соблюдение этих правил предотвратит поломку MCP сервера и обеспечит стабильную работу.**
