# Описание конфигурационного файла appsettings.json

## Ollama (настройки для работы с LLM)

| Ключ | Значение по умолчанию | Описание |
|------|------------------------|----------|
| BaseUrl | http://localhost:11434 | URL для подключения к Ollama API |
| Model | phi3:mini | Основная модель для генерации ответов |
| EmbeddingModel | nomic-embed-text | Модель для генерации эмбеддингов (векторных представлений текста) |
| TimeoutSeconds | 60 | Таймаут для запросов к Ollama API (в секундах) |
| Temperature | 0.7 | Температура генерации ответов (0 = детерминированное, 1 = максимально случайное) |
| MaxTokens | 500 | Максимальное количество токенов в ответе LLM |

## Indexer (настройки для индексации документов)

| Ключ | Значение по умолчанию | Описание |
|------|------------------------|----------|
| SupportedExtensions | [".txt", ".md", ".cs", ".js", ".ts", ".json", ".yaml", ".rst"] | Список поддерживаемых расширений для индексации |
| MaxFileSizeMB | 10 | Максимальный размер файла для индексации (в мегабайтах) |
| SkipLockedFiles | true | Пропускать заблокированные файлы при индексации |

## VectorStore (настройки для векторного хранилища ChromaDB)

| Ключ | Значение по умолчанию | Описание |
|------|------------------------|----------|
| Type | chromadb | Тип векторного хранилища |
| ConnectionString | http://localhost:8000 | URL для подключения к ChromaDB |
| CollectionName | documents | Имя коллекции в ChromaDB для хранения документов |
| ChunkSize | 1000 | Размер фрагмента при разборе документов на части (в символах) |
| ChunkOverlap | 200 | Перекрытие фрагментов документов (в символах) для сохранения контекста |

## RAG (настройки для системы Retrieval-Augmented Generation)

### Общие параметры

| Ключ | Значение по умолчанию | Описание |
|------|------------------------|----------|
| MaxChunks | 5 | Максимальное количество фрагментов для включения в контекст |
| MinRelevanceScore | 0.7 | Минимальный порог релевантности для включения в контекст |
| MaxContextTokens | 2000 | Максимальное количество токенов в контексте для генерации ответа |
| IncludeMetadataInContext | true | Включать метаданные (источник, путь) в контекст для генерации ответа |
| Temperature | 250.0 | Температура для экспоненциальной нормализации score (чем выше, тем мягче нормализация) |
| EnableAnswerGeneration | true | Включить генерацию ответа на основе найденных документов (если false, возвращается только список документов) |
| EnableHallucinationCheck | true | Включить проверку на галлюцинации (информацию не из контекста) |
| EnableRegeneration | true | Включить регенерацию ответа при галлюцинациях |

### Retry (настройки для retry-цикла)

| Ключ | Значение по умолчанию | Описание |
|------|------------------------|----------|
| MaxRetries | 2 | Максимальное количество попыток поиска |
| MinRelevantCount | 1 | Минимальное количество релевантных документов для успешного поиска |
| EnableQueryRewrite | true | Включить автоматическое перезапись запроса перед первым поиском |
| ScoreBoostPerRetry | 0.1 | Снижение порога релевантности при каждой дополнительной попытке |

### Hallucination (настройки для проверки на галлюцинации)

| Ключ | Значение по умолчанию | Описание |
|------|------------------------|----------|
| Enabled | true | Включить проверку на галлюцинации |
| MaxRegenerations | 1 | Максимальное количество регенераций ответа |
| ConfidenceThreshold | 0.7 | Порог уверенности для признания ответа основанным на контексте |
| UseDetailedCheck | false | Использовать детальную проверку по предложениям |

## Logging (настройки логирования)

### LogLevel (уровни логирования)

| Ключ | Значение по умолчанию | Описание |
|------|------------------------|----------|
| Default | Information | Уровень логирования для всех категорий |
| Microsoft | Warning | Уровень логирования для Microsoft |
| Microsoft.Hosting.Lifetime | Information | Уровень логирования для lifetime |
| McpRag | Information | Уровень логирования для McpRag |

### File (настройки для лог-файлов)

| Ключ | Значение по умолчанию | Описание |
|------|------------------------|----------|
| Path | logs/app- | Путь для записи логов в файл |
| FileSizeLimitBytes | 10485760 | Максимальный размер одного лог-файла (10MB) |
| RetainedFileCountLimit | 30 | Максимальное количество сохраняемых лог-файлов |
| RollingInterval | Day | Интервал для создания новых лог-файлов (по дням) |
